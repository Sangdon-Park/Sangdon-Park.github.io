submission.csv를 자기 이름 한글 파일명으로 바꿉니다. 영문 팀명은 쓰지 않습니다. 예: 박상돈.csv

5

비공개 채널에 업로드

예측 CSV는 공개 GitHub PR, 공개 issue, 공개 comment에 올리지 않습니다. 강의 공지에 안내된 비공개 제출 채널에 박상돈.csv처럼 한글 이름 파일명으로 업로드합니다.

6

채점 결과 확인

유효한 제출은 숨겨진 정답 label로 채점됩니다. 공개 리더보드에는 순위와 총점만 반영되며, 제출 CSV와 데이터셋별 세부 점수는 공개하지 않습니다.

필독 공지

점수 튜닝보다 검증 설계가 중요한 대회입니다

  • 제출은 GitHub 계정 기준 한국 시간 하루 3회까지만 채점됩니다.
  • 공개 GitHub PR은 예측값이 노출되므로 채점하지 않습니다.
  • 리더보드는 숨겨진 정답 label 기준의 총점만 공개합니다.
  • 두 데이터셋을 모두 제출해야 하며, 한쪽만 잘 맞추는 전략은 최종 점수에서 불리합니다.
  • 데이터에는 누락값, 범주형 shift, 시점 변화, decoy feature가 포함되어 있습니다.
  • 모델 종류는 제한하지 않습니다. 단, 제출 파일은 예측 CSV 하나만 받습니다.

Submission

제출 파일 형식

dataset,id,prediction
conversion,conversion_00901,0.7312
credit,credit_00908,0.1844

제출 파일명은 자기이름.csv입니다. 예: 박상돈.csv. 하나의 CSV 안에 conversion과 credit 예측 row가 모두 들어가야 합니다. 공개 저장소에는 업로드하지 않습니다.

Scoring

점수 계산식

0.5 × 고객 전환 ROC AUC + 0.5 × 신용 위험 ROC AUC

두 과제의 scale과 난이도가 다르기 때문에 단일 모델보다 데이터셋별 검증 전략과 calibration이 중요합니다.

Submit Checklist

제출 전 확인할 것

  • 파일명이 정확한가: 자기이름.csv 형식이며 파일명이 한글 이름인가
  • 컬럼이 정확한가: dataset,id,prediction
  • sample_submission.csv와 row 수가 같은가
  • conversion, credit 두 데이터셋이 모두 포함되어 있는가
  • prediction 값이 모두 0 이상 1 이하인가
  • 공개 GitHub PR, issue, comment에 예측 CSV를 올리지 않았는가
  • 하루 3회 제한을 넘기지 않았는가

Rules

운영 규칙

허용

전처리, feature engineering, AutoML, ensemble, 외부 오픈소스 라이브러리 사용을 허용합니다.

제한

GitHub 계정 기준 하루 3회 제출 제한이 있습니다. 같은 참가자는 하나의 한글 이름 파일명을 유지해야 합니다.

금지

평가 row를 수작업으로 라벨링하거나, 다른 팀의 제출값을 복사하거나, 제출 제한을 우회하거나, 예측 CSV를 공개하는 행위는 금지합니다.

확정

유효한 제출은 비공개 채점 뒤 반영됩니다. 정답 label과 제출 CSV는 공개 저장소에 저장하지 않고, 이 페이지에는 총점과 순위만 공개합니다.

FAQ

자주 하는 실수

문제 1, 문제 2를 파일 두 개로 제출하나요?
아닙니다. 박상돈.csv 하나만 제출합니다. 그 파일 안에 conversion row와 credit row를 모두 넣습니다.
test 파일에 target이 없어요.
정상입니다. conversion_test.csv와 credit_test.csv의 dataset, id에 맞춰 probability를 제출하세요.
한 데이터셋만 제출해도 되나요?
안 됩니다. sample_submission.csv의 모든 row를 포함해야 채점됩니다.
순위표 점수가 공식 점수인가요?
네. 숨겨진 정답 label로 계산한 공식 점수입니다. target 값, 제출 CSV, 데이터셋별 세부 점수는 공개하지 않습니다.
